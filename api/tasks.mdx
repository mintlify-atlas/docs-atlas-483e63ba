GameAP uses two distinct task types: daemon tasks and server scheduled tasks. Daemon tasks are background jobs dispatched to node daemons (such as starting or reinstalling a server) — they are created automatically by control operations and are read-only from the API. Server scheduled tasks are user-defined cron-like jobs that run commands against a game server on a repeating schedule.

## Daemon tasks

Daemon tasks are created automatically when you call control endpoints like `/start`, `/stop`, or `/install`. Use these endpoints to monitor the progress and output of those operations.

<Note>
  Listing daemon tasks and retrieving task output requires an administrator account or a PAT with the `admin:gdaemon-task:read` ability. Getting a single task by ID is accessible to users who have access to the associated server.
</Note>

<AccordionGroup>
  <Accordion title="GET /api/gdaemon_tasks — list daemon tasks">
    Returns a paginated list of all daemon tasks, ordered by creation time.

    ```bash
    curl "http://<host>:8025/api/gdaemon_tasks?page=1&per_page=20" \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```

    **Query parameters:**

    <ParamField query="page" type="integer" default="1">
      Page number.
    </ParamField>
    <ParamField query="per_page" type="integer" default="15">
      Results per page.
    </ParamField>
    <ParamField query="sort" type="string">
      Field to sort by.
    </ParamField>

    **Response (200):** Paginated list of daemon task objects.

    ```json
    {
      "current_page": 1,
      "data": [
        {
          "id": 123,
          "task": "start",
          "status": "success",
          "server_id": 6,
          "dedicated_server_id": 1,
          "created_at": "2026-04-22T14:00:00Z"
        }
      ],
      "last_page": 5,
      "per_page": 15,
      "total": 72
    }
    ```
  </Accordion>

  <Accordion title="GET /api/gdaemon_tasks/{id} — get daemon task">
    Returns the details for a single daemon task by ID. Accessible to users with access to the server associated with the task.

    ```bash
    curl http://<host>:8025/api/gdaemon_tasks/123 \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```

    **Response (200):** Daemon task object.

    <ResponseField name="id" type="integer" required>
      Task ID.
    </ResponseField>
    <ResponseField name="task" type="string" required>
      Task type, such as `start`, `stop`, `install`, `reinstall`, or `update`.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Current task status: `waiting`, `working`, `success`, or `error`.
    </ResponseField>
    <ResponseField name="server_id" type="integer">
      The game server this task is associated with (if applicable).
    </ResponseField>
    <ResponseField name="dedicated_server_id" type="integer" required>
      The node (dedicated server) where this task runs.
    </ResponseField>
    <ResponseField name="cmd" type="string">
      The command that was or will be executed by the daemon.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp when the task was created.
    </ResponseField>
    <ResponseField name="updated_at" type="string">
      ISO 8601 timestamp of the last status update.
    </ResponseField>
  </Accordion>

  <Accordion title="GET /api/gdaemon_tasks/{id}/output — get task output">
    Returns the console output captured from the daemon while executing the task. Useful for diagnosing failed installs or updates.

    Requires an administrator account or the `admin:gdaemon-task:read` ability.

    ```bash
    curl http://<host>:8025/api/gdaemon_tasks/123/output \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```

    **Response (200):**

    ```json
    {
      "output": "Downloading game files...\nInstallation complete.\n"
    }
    ```

    <ResponseField name="output" type="string">
      The raw text output from the daemon task execution.
    </ResponseField>
  </Accordion>
</AccordionGroup>

## Polling for task completion

Daemon tasks run asynchronously. After dispatching a control operation (start, stop, install, etc.), poll the task endpoint until `status` is `success` or `error`.

```bash
# 1. Start the server, capture the task_id
TASK_ID=$(curl -s -X POST http://<host>:8025/api/servers/6/start \
  -H "Authorization: Bearer YOUR_TOKEN" | jq -r '.task_id')

# 2. Poll until done
while true; do
  STATUS=$(curl -s http://<host>:8025/api/gdaemon_tasks/$TASK_ID \
    -H "Authorization: Bearer YOUR_TOKEN" | jq -r '.status')
  echo "Status: $STATUS"
  [ "$STATUS" = "success" ] || [ "$STATUS" = "error" ] && break
  sleep 3
done
```

## Server scheduled tasks

Server scheduled tasks are cron-like jobs you define per game server. They run commands (such as restarting the server or sending an announcement) on a schedule. Managing scheduled tasks requires the `server:tasks-manage` PAT ability.

<AccordionGroup>
  <Accordion title="GET /api/servers/{server}/tasks — list server tasks">
    Returns all scheduled tasks configured for a specific server.

    ```bash
    curl http://<host>:8025/api/servers/6/tasks \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```

    **Response (200):** Array of server task objects.
  </Accordion>

  <Accordion title="POST /api/servers/{server}/tasks — create scheduled task">
    Creates a new scheduled task for the server.

    ```bash
    curl -X POST http://<host>:8025/api/servers/6/tasks \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "command": "restart",
        "cron": "0 4 * * *",
        "enabled": true
      }'
    ```

    <ParamField body="command" type="string" required>
      The command or action to perform (e.g., `restart`, `say Server restarting soon`).
    </ParamField>
    <ParamField body="cron" type="string" required>
      Cron expression defining when the task runs (e.g., `0 4 * * *` for 4:00 AM daily).
    </ParamField>
    <ParamField body="enabled" type="boolean" default="true">
      Whether the task is active.
    </ParamField>

    **Response (200):** The created server task object.
  </Accordion>

  <Accordion title="PUT /api/servers/{server}/tasks/{id} — update scheduled task">
    Updates an existing scheduled task.

    ```bash
    curl -X PUT http://<host>:8025/api/servers/6/tasks/10 \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"cron": "0 6 * * *", "enabled": true}'
    ```

    **Response (200):** Updated server task object.
  </Accordion>

  <Accordion title="DELETE /api/servers/{server}/tasks/{id} — delete scheduled task">
    Deletes a scheduled task.

    ```bash
    curl -X DELETE http://<host>:8025/api/servers/6/tasks/10 \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```

    **Response (200):** `{"status": "ok"}`
  </Accordion>
</AccordionGroup>
