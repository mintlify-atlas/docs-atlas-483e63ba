The fastest way to get GameAP running is with Docker Compose. This guide walks you through launching the panel, connecting a node with the GameAP Daemon, and creating your first game server.

<Note>
  GameAP listens on port **8025** by default. Make sure this port is open on your host machine before proceeding.
</Note>

<Steps>
  <Step title="Start GameAP with Docker Compose">
    Clone the repository and start all services with a single command. The default `docker-compose.yml` starts GameAP with PostgreSQL and Redis.

    ```bash
    git clone https://github.com/gameap/gameap.git
    cd gameap
    docker-compose up -d
    ```

    This starts three containers:

    - `gameap` — the web panel, exposed on port 8025
    - `gameap-postgres` — PostgreSQL 17 database
    - `gameap-redis` — Redis 7 cache

    The full `docker-compose.yml` from the repository:

    ```yaml
    services:
      gameap:
        image: gameap/gameap:latest
        build:
          context: .
          dockerfile: Dockerfile
          args:
            VERSION: dev
            BUILD_DATE: ${BUILD_DATE:-2025-01-01T00:00:00Z}
        container_name: gameap
        entrypoint: ["/bin/sh", "-c", "mkdir -p /var/lib/gameap/files && /usr/bin/gameap"]
        ports:
          - "8025:8025"
        environment:
          HTTP_HOST: 0.0.0.0
          HTTP_PORT: 8025
          DATABASE_DRIVER: postgres
          DATABASE_URL: postgres://gameap:gameap@postgres:5432/gameap?sslmode=disable
          ENCRYPTION_KEY: ${ENCRYPTION_KEY:-change-me-in-production}
          AUTH_SECRET: ${AUTH_SECRET:-change-me-in-production}
          CACHE_DRIVER: redis
          CACHE_REDIS_ADDR: redis:6379
          FILES_DRIVER: local
          FILES_LOCAL_BASE_PATH: /var/lib/gameap/files
          LOGGER_LEVEL: ${LOGGER_LEVEL:-info}
        volumes:
          - gameap-storage:/var/lib/gameap
        depends_on:
          postgres:
            condition: service_healthy
          redis:
            condition: service_healthy
        networks:
          - gameap-network
        restart: unless-stopped
        healthcheck:
          test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:8025/api/health"]
          interval: 30s
          timeout: 3s
          start_period: 10s
          retries: 3
    
      postgres:
        image: postgres:17-alpine
        container_name: gameap-postgres
        environment:
          POSTGRES_DB: gameap
          POSTGRES_USER: gameap
          POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-gameap}
        volumes:
          - gameap-postgres-data:/var/lib/postgresql/data
        networks:
          - gameap-network
        restart: unless-stopped
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U gameap"]
          interval: 10s
          timeout: 5s
          retries: 5
    
      redis:
        image: redis:7-alpine
        container_name: gameap-redis
        command: redis-server --appendonly yes
        volumes:
          - gameap-redis-data:/data
        networks:
          - gameap-network
        restart: unless-stopped
        healthcheck:
          test: ["CMD", "redis-cli", "ping"]
          interval: 10s
          timeout: 5s
          retries: 5
    
    volumes:
      gameap-storage:
        driver: local
      gameap-postgres-data:
        driver: local
      gameap-redis-data:
        driver: local
    
    networks:
      gameap-network:
        driver: bridge
    ```

    To configure secrets and other options before starting, create a `.env` file in the same directory:

    ```bash
    # Security - REQUIRED IN PRODUCTION
    ENCRYPTION_KEY=change-me-to-random-32-char-key
    AUTH_SECRET=change-me-to-random-32-char-secret
    
    # Database
    DATABASE_DRIVER=postgres
    DATABASE_URL=postgres://gameap:gameap@postgres:5432/gameap?sslmode=disable
    POSTGRES_PASSWORD=gameap
    
    # Cache
    CACHE_DRIVER=redis
    CACHE_REDIS_ADDR=redis:6379
    
    # File storage
    FILES_DRIVER=local
    FILES_LOCAL_BASE_PATH=/var/lib/gameap/files
    
    # Logging
    LOGGER_LEVEL=info
    
    # Authentication
    AUTH_SERVICE=paseto
    ```

    <Warning>
      The default `ENCRYPTION_KEY` and `AUTH_SECRET` values in the repository are placeholders. **Always set strong, randomly generated values in production.** These secrets protect your session tokens and encrypted data. If you change them after the panel has been used, existing sessions and encrypted records will be invalidated.
    </Warning>
  </Step>
  <Step title="Access the panel">
    Once the containers are healthy, open your browser and navigate to:

    ```text
    http://localhost:8025
    ```

    On first launch, GameAP will prompt you to create an administrator account. Complete the setup wizard to configure the initial admin user and verify the database connection is working.

    To confirm the containers are running and healthy:

    ```bash
    docker-compose ps
    ```

    To follow panel logs in real time:

    ```bash
    docker-compose logs -f gameap
    ```
  </Step>
  <Step title="Add a node (daemon setup)">
    Each machine that will host game servers needs the **GameAP Daemon** installed. The daemon is a lightweight agent that the panel communicates with to manage game server processes.

    In the web UI:

    1. Navigate to **Nodes** in the sidebar.
    2. Click **Add node**.
    3. Fill in the node's hostname or IP address and the daemon port (default: `31717`).
    4. Copy the generated setup script and run it on the target machine as root.

    The setup script installs the GameAP Daemon, configures it to connect back to your panel, and starts it as a system service. After the node is registered, it will appear as **online** in the Nodes list.

    For more details on daemon installation and configuration options, see the [nodes guide](/guides/nodes).
  </Step>
  <Step title="Create your first game server">
    With a node online, you can launch your first game server:

    1. Navigate to **Game Servers** in the sidebar.
    2. Click **Create server**.
    3. Select the node where the server will run.
    4. Choose a game from the catalog (for example, Minecraft or Counter-Strike 2).
    5. Configure the server name, ports, and any game-specific settings.
    6. Click **Create** — GameAP will install the server files on the node and start the process automatically.

    Once created, you can start, stop, restart, view logs, and manage files for the server directly from the panel.

    For more detail on server options and advanced configuration, see the [game servers guide](/guides/game-servers).
  </Step>
</Steps>
